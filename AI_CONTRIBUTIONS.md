## How This Project Was Built

Zokai Station was built using a **Human-Directed, AI-Accelerated** workflow. All code in this repository was generated by enterprise AI tools under continuous human architectural direction, review, and refinement.

### Tools Used

| Tool | Role | License Impact |
|---|---|---|
| **Google Antigravity** (Pro/Ultra) | Primary code generation, architectural planning, debugging | User retains full ownership per commercial ToS |
| **Anthropic Claude** (via Kilo Code) | Code generation, in-IDE agent tasks, refactoring | User retains full ownership per commercial ToS |

### The Workflow

```
Human (MKN)                          AI Tools
─────────────                        ────────
Design architecture          ───►    Generate implementation
Review output                ◄───    Propose code
Accept / Reject / Refine     ───►    Iterate

```

Every commit in this repository represents a **human review-accept-commit cycle**, not automated push.

### What the Human Owns

Under current (2026) copyright standards, copyright on AI-assisted works requires demonstrable **"Selection and Arrangement"** — substantive human creative decisions. This project's human contributions include:

- **Architecture**: 4-network Docker Compose isolation, Sleeper Container pattern, MCP Bridge Proxy design
- **Selection**: Choosing which AI output to keep vs. reject across 223+ commits
- **Integration**: Wiring 20+ services together (volume mirroring, secret injection, tier isolation)
- **Curation**: 50+ architectural decisions documented in `memory-bank/decision-log.md`
- **Specification**: Formal specs in `docs/specs/` defining behavior before AI generation
- **Review**: 15+ retrospectives in `docs/retrospectives/` capturing lessons learned
